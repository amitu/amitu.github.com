{% extends "page.html" %} {% block title %}smarturls{% endblock %} {% block comment %}{% endblock %} {% block page_content %} {% filter md %} smarturls comes with a library of regularly used regular expression patterns to easy regex creation for django urls. The library can be extended by using django setting system. #### Install smarturls :::bash $ pip smarturls #### Example usage This is how normal django urls.py looks like: :::python from django.conf.urls.defaults import * urlpatterns = patterns('', ('^book/(?P\d+)/$', 'some.view'), ('^author/(?P[-\w]+)/$', 'some.other.view'), ('^year/(?P\d{4,4})/$', 'year.view'), ('^year/(?P\d{4,4})/(?P\w+)/$', 'month.view'), ) This is how it looks like with smarturls: :::python from django.conf.urls.defaults import * from smarturls import surl urlpatterns = patterns('', surl('/book//', 'some.view'), surl('/author//', 'some.other.view'), surl('/year//', 'year.view'), surl('/year///', 'month.view'), ) #### Custom patterns Existing patterns can be overwritten and new ones can be defined by defingin the django setting SURL_REGEXERS. Eg, in settings.py: :::python SURL_REGEXERS = { "slug": "\w[-\w]*", # overwrite slug: always start with alphanumeric "username": "\w+" # now you can use /author// etc } #### Default Patterns By default smarturls comes with the following patterns: * int: \d+ * int2: \d{2,2} * int4: \d{4,4} * word: \w+ * slug: \[\w-\]+ * digit: \d{1,1} * username: \[\w.@+-\]+ * base64: [0-9a-zA-Z\-_]+={0,2} * uuid: "[A-Fa-f0-9]{8}-?[A-Fa-f0-9]{4}-?4[A-Fa-f0-9]{3}-?[89abAB][a-fA-F0-9]{3}-?[a-fA-F0-9]{12}" * [ekey](http://github.com/amitu/django-encrypted-id): "[0-9a-zA-Z\-_]+\.{0,2}" #### Contribute Do you repeatedly use some URL pattern that everyone may want to use?
